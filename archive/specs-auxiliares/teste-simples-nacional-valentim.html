Simples Nacional — Ativado ✅

Empresa: DAP DISTRIBUIDORA · Ativação: 24/04/2026
Enviado por: Eduardo · Para: Valentim
TL;DR — resumo em 3 linhas
  • A feature nova que identifica se o cliente é do Simples e escolhe a natureza certa do Tiny tá ligada.
  • Funciona só pra DAP. Cântico continua igual.
  • Preciso que você faça 1 ou 2 testes rápidos pra confirmar que tá indo pro Tiny certinho.

1. O que foi ligado

Antes, cada operação do ProSeller tinha 1 código Tiny. Agora, pra DAP, a Venda de mercadorias e a Bonificação têm 2 códigos — um para quando o cliente é do Simples, outro quando não é. O sistema escolhe sozinho baseado no CNPJ do cliente consultado na Receita Federal.

Os códigos que você me mandou ontem já estão cadastrados:

Operação ProSeller Código se NÃO-Simples Código se Simples
Venda de mercadorias 781495108 781284632
Bonificação 781496980 657287750

Devolução, Remessa e demais operações continuam com 1 código só — não tem toggle.

2. Como o teste funciona

Você pega um cliente PJ da DAP e cria um pedido nele normalmente. O ProSeller consulta a Receita Federal, descobre se o CNPJ é Simples, envia o pedido pro Tiny com o código correto e pronto.

O que eu preciso você confirmar: o pedido chegou no Tiny com a natureza de operação certa?

3. Roteiro (rápido)

Teste A — Cliente que você SABE que é do Simples

  1. Abre o ProSeller normalmente.
  2. Escolhe um cliente PJ da DAP que você sabe que é optante do Simples.
  3. Cria um pedido simples (1 item, qualquer valor pequeno — depois pode cancelar).
  4. Dispara o envio pro Tiny como você faria normalmente.
  5. Abre o Tiny e localiza o pedido que acabou de chegar.
  6. Confere a "Natureza de Operação" do pedido:
    • Se for Venda → deve estar com código 781284632 Simples
    • Se for Bonificação → deve estar com 657287750 Simples

Teste B — Cliente que você SABE que NÃO é do Simples

  1. Mesmos passos 1–5 acima, mas escolhendo um cliente da DAP que não é optante do Simples.
  2. Confere no Tiny:
    • Se for Venda → deve estar com 781495108 não-Simples
    • Se for Bonificação → deve estar com 781496980 não-Simples

4. O que me mandar

Pode ser bem simples, um zap do tipo:

"Testei Venda com cliente Simples — chegou no Tiny com 781284632. ✅"

"Testei Bonificação cliente não-Simples — chegou com 781496980. ✅"

Se quiser mandar print do pedido no Tiny junto, ajuda — dá pra ver a natureza ali direto.

5. Se algo parecer errado

Qualquer coisa estranha, me avisa imediatamente. Exemplos:
  • Pedido chegou no Tiny com natureza de venda normal quando era pra ser Simples.
  • Pedido não chegou no Tiny.
  • Erro ao salvar o pedido no ProSeller.
  • Qualquer comportamento diferente do que você tá acostumado.

A feature tem uma chave liga/desliga no servidor. Se der problema, eu desligo em segundos e o sistema volta exatamente ao comportamento de ontem. Você não perde nada.

6. Observação sobre consultas à Receita

A consulta é feita a cada pedido enviado pra DAP (pra garantir dado fresco, já que cliente pode sair do Simples fora das janelas anuais). Pra Cântico, como ela não tem o toggle ligado, o sistema pula a consulta — não desperdiça.

A API que consultamos (ReceitaWS pública) tem limite de 3 consultas por minuto. Se bater limite num pico, o pedido vai com o código padrão não-Simples — mais conservador. Se isso virar frequente, a gente contrata o plano pago (é rápido).
